# 期末任务跟踪助手

## 📋 概述

**服务对象**：大学生（正在准备期末考试、作业、论文的学生）

**具体问题**：期末期间任务繁重且时间紧迫，学生常常面临以下痛点：
- 任务分散在各个课程平台，难以统一管理
- 难以判断哪些任务最紧急，容易错过截止日期
- 难以合理规划每日时间，导致拖延或效率低下
- 课程安排与任务时间冲突，不知道如何平衡

---

## 📥 输入内容

| 输入类型 | 示例 |
|---------|------|
| 文本描述 | "我有个高数作业，截止6月20号" |
| 快捷指令 | "添加任务"、"查看进度"、"今日计划" |
| 任务信息 | 课程名称、任务类型、截止时间、优先级、预计工时 |
| 课程信息 | 课程名称、上课时间、上课地点 |
| 待办事项 | 内容、分类（学习/生活/社交）、优先级 |

---

## 📤 输出结果

| 输出类型 | 示例 |
|---------|------|
| 任务统计 | 按课程/状态/优先级分类的进度报告 |
| 风险预警 | 即将逾期任务提醒（3天内到期、高优先级未开始） |
| 今日计划 | 整合课程表+期末任务+待办事项的每日建议清单 |
| 操作确认 | 任务添加成功/更新成功等反馈 |

---

## ⚙️ 处理步骤

```
用户输入 → 意图识别 → 信息提取 → 工具调用 → 结果格式化 → 返回用户
```

1. **意图识别**：判断用户是想添加任务、查询进度、还是生成计划
2. **信息提取**：从用户描述中提取任务名称、课程、时间等关键信息
3. **工具调用**：通过飞书多维表格API执行CRUD操作
4. **结果格式化**：以简洁的emoji表格形式返回结果

---

## 🤖 模型任务（大模型）

| 任务 | 说明 |
|------|------|
| 意图理解 | 识别用户想要执行的操作类型 |
| 信息提取 | 从自然语言中提取结构化信息（日期、课程、优先级等） |
| 日期补全 | 将"截止6月20号"补全为完整日期（考虑年份） |
| 风险评估 | 根据DDL紧迫度和进度计算风险等级 |
| 话术生成 | 生成简洁友好的回复内容 |
| 选择式交互 | 生成选项卡片供用户选择 |

---

## 🔧 工具任务（工作流/代码/飞书）

| 工具 | 职责 |
|------|------|
| 飞书多维表格 | 持久化存储任务、课程表、待办事项 |
| get_current_time | 获取当前时间，用于日期处理 |
| add_task | 向飞书任务总览表添加记录 |
| update_task | 更新任务进度和状态 |
| get_all_tasks | 查询所有任务列表 |
| get_task_statistics | 统计任务完成情况 |
| generate_daily_plan_suggestion | 生成每日任务建议 |
| get_today_schedule | 获取今日综合安排 |
| get_course_schedule | 查询课程表 |
| add_todo | 添加代办事项 |

---

## ⚠️ 异常情况处理

| 场景 | 处理方式 |
|------|----------|
| 信息不完整 | 主动询问缺失字段（如未指定优先级/工时） |
| 日期模糊 | 使用当前时间补全年份，如"已过该日期则使用下一年" |
| 飞书API失败 | 返回友好提示："API暂时不可用，已记录你的需求" |
| 找不到对应字段 | 提示用户确认信息，并说明可能需要初始化表格 |
| 无任务数据 | 友好提示"暂无任务"，引导用户添加 |
